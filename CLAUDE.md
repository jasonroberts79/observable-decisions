# Observable Decisions

A full-stack decision management platform. React SPA frontend + Python FastAPI backend, deployed on GCP.

## Repo Layout

```
app/      React 19 + TypeScript + Vite + Tailwind CSS → Firebase Hosting
api/      Python 3.12 FastAPI + pytest → GCP Cloud Run (Docker)
infra/    Terraform (GCP: Cloud Run, Firebase, GCS, IAM)
.github/  Three GitHub Actions workflows: backend.yml, frontend.yml, infra.yml
```

## Running Locally

**Frontend** (runs on http://localhost:5173):
```bash
cd app
npm install
npm run dev
```

**Backend** (runs on http://localhost:8000):
```bash
cd api
uv sync
uv run uvicorn main:app --reload
```

When running locally without Firebase configured, the API accepts an `X-User-Email` header instead of a Bearer token.

## Tests

```bash
cd api
uv run pytest tests/ -v --tb=short
```

## Key Env Vars

`.env.local` at repo root and `app/` for Vite:

| Variable | Where | Purpose |
|---|---|---|
| `FIREBASE_PROJECT_ID` | `api/.env` | Enables Firebase token verification in API |
| `GCS_BUCKET_NAME` | `api/.env` | GCS bucket for decision storage (required for API to start) |
| `API_CORS_ORIGINS` | `api/.env` | Comma-separated allowed origins (default: `http://localhost:3000`) |
| `VITE_FIREBASE_*` | `app/.env` | Firebase client config (API key, project ID, etc.) |

Never commit `.env` or `.env.local`. These are gitignored.

## Architecture

- **Auth**: Firebase Authentication (Google + GitHub OAuth). The API verifies Firebase ID tokens via `firebase-admin`. Locally, `X-User-Email` header is the fallback when `FIREBASE_PROJECT_ID` is unset.
- **Storage**: All decision records stored as JSON objects in GCS (`users/{email}/{id}.json`). The `StorageBackend` abstraction is in `api/storage/base.py`; the GCS implementation is in `api/storage/gcs.py`.
- **Data models**: Zod schemas live in `app/src/lib/decisions/schema.ts` (frontend). Pydantic models in `api/models.py` (backend). All JSON uses camelCase (`model_dump(by_alias=True)`).
- **Share links**: Signed GCS URLs with 7-day expiry, generated by `POST /api/decisions/{id}/share`.

## Conventions

- **Never manually edit** `uv.lock` or `package-lock.json`.
- Frontend components: `app/src/components/` (reusable), `app/src/pages/` (route-level).
- All API endpoints are prefixed `/api/` and require `Authorization: Bearer <firebase-token>`.
- Infra changes in `infra/` auto-apply to production on merge to `main` via `infra.yml`.

## Deployment

| Component | Trigger | Target |
|---|---|---|
| Frontend | Push to `main` touching `app/**` | Firebase Hosting |
| Backend | Push to `main` touching `api/**` | GCP Cloud Run |
| Infra | Push to `main` touching `infra/**` | Terraform apply |

CI uses Workload Identity Federation — no long-lived GCP keys in GitHub secrets.
